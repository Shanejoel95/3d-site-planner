# Plan Template

Assemble the final plan in this exact section order. Fill every section; if a
section has no data, state that and why (e.g. "No website found — see opportunity
below") rather than omitting it. Keep it tight: bullets over paragraphs, every
recommendation traceable to a finding. Replace bracketed placeholders.

---

# 3D Website Plan — [Client Name]
*Prepared [date] · Primary goal: [call / booking / purchase / lead] · Recommended
stack: [stack] · Mode: [A — preserve existing site / B — build from scratch]*

> **Mode A (existing site):** the client's real copy, assets, and structure are
> preserved and animated, not rewritten. Any proposed content change is listed
> explicitly for approval.

## 1. Executive summary
3–5 sentences: who the client is, the core opportunity, the recommended 3D approach,
and the headline of the plan. Written so the client could read just this and get it.

## 2. Client snapshot
- **Business:** what they do, where, for whom.
- **Current presence:** website (yes/no + state), social, video.
- **Primary conversion goal:** [...]
- **Sources used:** list what was reviewed (site, reviews, articles, etc.). Note
  anything that couldn't be found.

## 3. Research findings
Evidence gathered, by source (website, reviews, press, video, photography, social,
competitors). Paraphrase reviews/quotes — never paste long verbatim text. Cite
ratings/counts where real.

## 4. Analysis
- **Scorecard** — the 10-dimension rubric table (dimension · score 1–5 · note).
- **Highlights** — strengths/assets to amplify (each tied to a finding).
- **Drawbacks / gaps** — what's missing or working against them (each paired with the
  action that addresses it).
- **Audience & positioning** — target customer and how to stand out.
- **Constraints** — technical, budget, timeline, regulatory.

## 5. Recommended 3D approach
- **Recommendation:** [stack] — and *why it fits this client specifically*.
- **Tradeoffs accepted.**
- **Fallback approach** if budget/timeline tightens.
- **Non-3D fallback experience** (reduced-motion, low-power, crawlers).

## 6. Build brief
- **Sitemap** — pages/sections + purpose.
- **3D scene concepts** — per key page/section: what's in it, the interaction, what
  it communicates, how it drives the goal.
- **Signature hero concept** — the one defining moment (cite the inspiration refs).
- **Motion & animation system** — the buildable recipe from Phase 3b: engine +
  libraries, scroll choreography (what scrubs to what), each signature effect mapped to
  its technique, shaders/materials/post-processing, and a scroll-timeline table if
  useful. Specific enough to install and build from; cite the reference per moment.
- **Copy** — *Mode A:* the client's existing copy reused verbatim, mapped to scenes,
  plus a separate flagged list of any proposed edits. *Mode B:* draft voice, messages,
  headlines (grounded in findings), marked draft-for-approval.
- **Asset list** — models, textures, photography, video, icons, fonts: *Mode A* reuses
  the client's existing assets first (logo, colors, fonts, images, video), flagging
  only genuinely new ones; *Mode B* is have / source / create (and how).
- **Inspiration references** — 2–4 specific examples from the galleries and what each
  contributes to form/motion.
- **Interactions & motion** — scroll, hover, transitions.
- **Performance budget** — load targets, asset ceilings, mobile strategy.
- **Accessibility & SEO** — reduced-motion, keyboard, crawlable content, metadata,
  structured data, analytics.

## 7. Phased roadmap
For each phase (Discovery → Design/Prototype → 3D Asset Production → Build → QA &
Performance → Launch → Post-launch): deliverables · rough effort/duration ·
dependencies · key risks. Flag the single biggest timeline risk.

## 8. Open questions & assumptions
What you assumed (e.g. budget tier, deadline) and what you need from the client to
finalize.
